 一、总评

  这个后端已经不是“只写了 README 的 AI 项目”，代码里有真实 FastAPI SSE 管道、商品数据、SQLModel 表、pgvector 路径、embedding/rerank/LLM provider 封装、
  商品卡结构化返回和最小购物车闭环。最大优点是工程结构清楚，API -> Runtime -> Service -> Repo 基本成立，核心链路能在默认 fallback 模式跑出完整 SSE 事件
  和商品卡。

  最大风险是：当前可验证环境没有通过 live RAG 验真。smoke_live_rag 失败，当前本地索引是 1292 chunks / 16 维 embedding，不是要求的 1024 维；demo_smoke
  虽然 6/6 通过，但过程中真实 LLM/embedding/rerank 连接失败后走了 deterministic fallback。如果我是评委，我会相信这里有真实 RAG 实现路径，但不会相信“当
  前提交状态拿来就能稳定 live 答辩”。

  答辩最可能被追问：为什么当前 live smoke 不通过、为什么常见问题会先澄清而不是推荐、为什么多轮“跑鞋 -> 轻量 -> 500 以内”会丢失品类上下文。

  四、接口与链路核查

  核心链路：

  用户请求 -> POST /chat/stream -> run_intent -> slot check -> run_criteria -> retrieve_with_evidence -> product cards -> recommendation text ->
  decision -> SSE done。

  核心接口：

  - POST /chat/stream，请求字段见 backend/src/types/schemas.py:20，响应为 SSE：backend/src/api/chat.py:14。
  - POST /chat/cancel，支持本进程 token + DB cancel request。
  - POST /upload/image，multipart 可真实保存；JSON legacy 是占位。
  - POST /feedback，写入反馈。
  - GET /cart/{session_id}，查看购物车。
  - GET /admin/eval/*、GET /admin/observability/*，评测与调试。

  SSE 支持真实流式 StreamingResponse，事件类型含 thinking / clarification / criteria_card / text_delta / product_card / cart_action / final_decision /
  done / error：backend/src/types/sse_events.py:116。商品卡有 product_id/name/price/image_url/category/brand/evidence，客户端可直接渲染。每个事件有
  session_id/turn_id/seq/event_id。错误有 code/message/retryable。

  五、RAG 真实性判断

  1. 商品数据是否真实入库：代码支持，raw 数据 100 条，本地 DB 有 1292 chunks；商品真相源来自 backend/src/repos/products.py:34。
  2. 是否真的 embedding：代码支持 live embedding，但当前可验证索引是 16 维 fallback，不是 1024 维 live。
  3. 是否真的向量检索：Postgres 下有 pgvector <=> 查询：backend/src/repos/documents.py:61；默认 SQLite 走 JSON/Python fallback。
  4. 是否把检索结果注入 LLM：注入了 product payload；没有注入 evidence chunks，证据只在 product_card 层绑定。
  5. 是否有结构化过滤：有 category、budget、ingredient/feedback avoid；品牌、产地、库存、优惠券等不足。
  6. 是否有防幻觉机制：商品卡来自库内商品，decision winner 校验 ID；但自然语言 text_delta 仍可能被 live LLM 写出 unsupported claim。
  7. 是否可能推荐不存在商品：product_card 基本不会；LLM 文案仍有风险。
  8. 是否能解释来源：DB chunk 命中时有 source_id；fallback evidence 有时只是产品级 snippet，弱于真实 chunk citation。
