## 致命问题：审计文档的 3 个事实性错误

### 错误 A：regex 贪婪 bug 不存在

**审计声称**：`message_rules.py:177` 的 `_ADJUST_BUDGET_CAP_PATTERN` 用了 `.{0,3}` 吞噬数字，`_ADJUST_AVOID_PATTERN` 用了 `(.{1,8})` 吞噬后续子句。

**实际代码**（`backend/src/runtime/message_rules.py`）：

```python
# line 178 — 注意 [^0-9]，不是 .
_ADJUST_BUDGET_CAP_PATTERN = re.compile(r"预算[^0-9]{0,3}(\d+(?:\.\d+)?)")

# line 177 — 注意 [^，。！？；、,.!?;:\n]，不是 .
_ADJUST_AVOID_PATTERN = re.compile(r"不要([^，。！？；、,.!?;:\n]{1,8})")
```

`[^0-9]` 明确排除数字，`[^，。！？；、,.!?;:\n]` 明确排除标点分隔符。**两个 bug 要么已修复，要么从未存在。**

**连锁影响**：
- 审计 P0 建议"修复 regex 贪婪量词 bug"是不存在的任务
- `test_viewmodel_contract.py:138-140` 断言 `budget_max == 200.0` 是正确的，测试会通过
- 审计声称"测试断言 `budget_max==0.0`"是错的

**验证命令**：
