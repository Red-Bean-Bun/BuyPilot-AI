## 三、铁律 5 违规清单

### 违规 1：conftest.py mock 打在内部函数而非 I/O 边界 — ⏳ 待修复

- **严重度**：高
- **位置**：`conftest.py` → `mock_external_ai` fixture
- **违规**：monkeypatch 目标是 `llm_gateway._chat_completion`（内部函数），而非 `httpx.AsyncClient.post`（I/O 边界）
- **当前缓解**：新增的 3 个 HTTP 测试文件（`test_embedding_normal_paths.py` / `test_reranker_normal_paths.py` / `test_vl_embedding_contract.py`）通过覆盖 `mock_external_ai` fixture 绕过了这个问题
- **为什么还没修**：另一个 Agent 正在重写 `conftest.py`（改 SQLite → PostgreSQL），改 conftest 会与它冲突
- **修复方案**：等另一个 Agent 合并后，将 conftest mock 目标改为 `httpx.AsyncClient.post`

### 违规 2：test_image_embedding.py 同义反复测试 — 🔶 部分修复

- **严重度**：高
- **位置**：`test_image_embedding.py::test_vl_embedding_request_sends_correct_payload`
- **违规**：monkeypatch 替换了被测函数 `_vl_embedding_request`，然后验证被替换后的函数行为
- **缓解**：新增 `test_vl_embedding_contract.py`（6 测试）提供正确的 VL embedding 契约测试
- **待办**：删除或重写 `test_image_embedding.py` 中的同义反复测试

### 违规 3：test_image_embedding.py 复制实现逻辑 — 🔶 部分修复
