## 三、代码事实链路

文字版链路如下：

```text
用户请求
-> POST /chat/stream
-> FastAPI StreamingResponse
-> runtime.pipeline.chat_stream
-> 图片预分析 / VLM 分析
-> 意图识别 analyze_intent
-> 槽位澄清 / 多轮上下文合并
-> criteria 生成
-> pgvector chunk 召回 + SQL hard filter
-> rerank
-> evidence 绑定
-> LLM 推荐文案流式生成
-> product_card / criteria_card / text_delta / final_decision / cart_action SSE 事件
-> Android 结构化渲染
```

关键证据：

- `/chat/stream` 返回真实 SSE：`backend/src/api/chat.py:63-92`
- 商品 payload 与商品卡事件结构：`backend/src/types/sse_events.py:97-110`、`backend/src/types/sse_events.py:234-244`
- 检索入口 `retrieve_with_evidence`：`backend/src/services/retriever.py:87-180`
- pgvector 查询：`backend/src/repos/documents.py:75-137`
- hard filter：`backend/src/services/retriever.py:423-480`
- 推荐处理器会发商品卡：`backend/src/runtime/handlers.py:432-587`
- LLM task-oriented interface：`backend/src/services/llm_client.py:126-255`
- provider fallback 与观测记录：`backend/src/services/llm_gateway.py:58-152`
- 数据入库与 embedding：`backend/src/services/product_ingest.py:40-127`
- 多模态图片分析与图片 embedding：`backend/src/runtime/stages/multimodal.py:20-40`
- 购物车 handler：`backend/src/runtime/cart_handlers.py`
- 多商品对比 handler：`backend/src/runtime/compare_handlers.py`

## 六、主要优点

1. **不是 mock 项目。** 代码中存在真实 RAG 检索、embedding、pgvector、rerank、evidence 绑定与结构化 SSE。
2. **商品事实有边界。** Prompt 明确要求只解释传入商品，不得编造商品、价格、优惠、库存；代码也校验商品卡来自库内 `product_id`。
3. **结构化返回足够支撑 Android。** `product_card`、`criteria_card`、`final_decision`、`cart_action`、`compare_card` 都是稳定事件模型。
4. **多轮与反馈有真实状态。** 会保存上一轮 criteria、deck、product_ids、feedback，并可继续决策或替换候选集。
5. **加分项不是只有目录名。** 购物车 CRUD、对比卡、多模态图片分析与视觉 embedding 召回都有实现。

| 隐性规则 | 结论 | 说明 |
| --- | --- | --- |
| 是否编造商品、价格、优惠、库存 | 基本可防 | Prompt 和代码均限制候选商品，商业声明问题有专门拦截 |

| 检索结果与最终推荐是否可能不一致 | 风险较低 | 商品卡来自检索结果，LLM 只生成解释；最终 winner 受候选约束 |
| 是否有商品 ID | 有 | `ProductPayload.product_id` 必填 |
| 是否有会话 ID | 有 | 未传时后端生成 `sess_...` |
| 是否结构化返回 | 有 | 多类 SSE event model |
| 是否真实流式 | 有 | `StreamingResponse` + LLM stream + text_delta |
| 是否错误处理粗糙 | 中等风险 | pipeline 有 error event，但 live provider/DB 超时仍需压测 |
| 是否 README 好看但代码没实现 | 否 | 代码实现基本能对上 README |
| 是否泄漏 API Key | 已缓解 | 具体 `ADMIN_API_KEY` 已替换为占位符，仍建议轮换曾使用过的真实 key |
