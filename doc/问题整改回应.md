# BuyPilot 问题整改回应

更新时间：2026-05-28

本文档回应 `doc/问题记录.md` 中 0-9 共十个问题。整改口径按本次确认的产品决策执行：真实修后端契约，改为三段式导购链路；默认首轮只生成购买标准并等待确认；继续/收敛由 LLM intent 识别；候选商品阶段引入 `deck_id`，最终决策读取当前 deck 的反馈后再生成。

## 代码核实结论

整体与当前代码事实相符：三段式 SSE 状态机、`finish_reason`、`deck_id`、`continue` intent、字段来源标记、手机预算必问、不支持品类拦截均能在代码和测试中找到对应实现。

补充整改：原始数据不能修改，因此食品品类采用 taxonomy 层归一。raw/source category 原样保留；业务侧统一使用 `食品生活`；`食品饮料`、`酱油/调味品` 等通过 `normalize_category` 和 `infer_category_from_product_type` 归入 `食品生活`；检索硬过滤也按归一后的 category 比较，兼容旧库或旧 payload 中残留的 `食品饮料`。

## 整体整改结果

后端 SSE 主链路已调整为：

```text
Turn 1: criteria_card -> done(awaiting_criteria_confirmation)
Turn 2: product_card* -> done(awaiting_product_feedback, deck_id=...)
Turn 3: final_decision -> done(completed)
```

核心契约变更：

- `DoneEvent.finish_reason`：`awaiting_criteria_confirmation` / `awaiting_product_feedback` / `completed` / `cancelled` / `error`
- `CriteriaPayload.field_sources`：字段来源标记为 `user` / `inferred` / `history`
- `FeedbackRequest.deck_id`、`Conversation.deck_id`、`Feedback.deck_id`：将反馈绑定到当前候选 deck
- `IntentResult.intent` 增加 `continue`：由 LLM 判断用户是否确认继续或要求收敛
- 食品品类通过 taxonomy 层归一为 `食品生活`；raw 商品数据仍可保留 `食品饮料`，入库 metadata 记录 `source_category`
