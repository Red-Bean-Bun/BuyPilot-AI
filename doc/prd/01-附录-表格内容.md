## sheet-id: pjFAv8

**设计原则**

| 原则 | 说明 | 示例 |
|------|------|------|
| Agent 感知优先 | UI 必须让用户感知到"AI 在逐步思考并给出结论"，而非传统搜索列表 | thinking 动画 → 标准卡 → 推荐卡 → 决策卡，形成完整叙事链 |
| 证据可追溯 | 每条推荐必须有可查看的证据来源，不做黑盒推荐 | product_card 带 evidence_refs，点击"看证据"展开 Bottom Sheet |
| 流式渐进展示 | 内容到达即渲染，不等全部完成再一次性展示 | text_delta 逐块追加，product_card 到达即插入 SwipeDeck |
| 单一时间线 | 所有 AI 输出（文本、卡片、thinking）必须在同一条聊天流中，不分散到多个区域 | LazyColumn 统一承载，不另设独立推荐区 |
| 可中断可修正 | 用户随时可停止生成、修改标准、表达不喜欢，系统响应修正 | 停止按钮、criteria_patch、feedback action |
| 最小惊讶 | 不做大位移动画、不自动跳转、不弹窗打断用户阅读 | 卡片淡入，证据在底板展示而非跳转外链 |
| 三周可交付 | 所有设计决策以三周内可实现为前提，不追求视觉完美 | P0 只做核心链路，P1/P2 明确标注 |

## sheet-id: U0OTLF

**输入区行为规范**

| 状态 | 行为 | 说明 |
|------|------|------|
| 空闲态 (Idle) | 输入框可编辑，发送按钮置灰，图片按钮可点击 | 默认状态，placeholder 显示引导文案 |
| 输入中 (Composing) | 输入框可编辑，发送按钮高亮，图片按钮可点击 | 文本非空或已选图时激活发送 |
| 图片已选 (ImageAttached) | 输入框上方显示图片缩略图 + 删除按钮，发送按钮高亮 | 单图限制，点击缩略图可预览 |
| 生成中 (Streaming) | 输入框不可编辑，发送按钮替换为停止按钮，图片按钮禁用 | 停止按钮点击后立即关闭 SSE |
| 已取消 (Canceled) | 输入框恢复可编辑，发送按钮恢复，保留已生成内容 | 用户可继续追问 |
| 错误态 (Error) | 输入框可编辑，显示重试按钮（替代发送按钮位置） | 重试按钮重发上一条请求 |
| 澄清等待 (Clarifying) | 输入框可编辑，placeholder 变为"请回答上面的问题" | 收到 clarification 后进入 |

## sheet-id: NriNH0

**消息列表渲染策略**

| SSE 事件 | 渲染方式 | 节点策略 | 说明 |
|----------|----------|----------|------|
| `thinking` | 更新当前 AI 回复节点内的 thinking 动画气泡 | upsert `node_id`，不新增列表项 | phase/message 驱动阶段文案切换 |
| `text_delta` | 追加文本到当前 AI 回复节点 | 与 thinking 共享或使用独立 `node_id`，增量 append | done=true 时冻结该节点 |
| `clarification` | 插入澄清卡节点 | 新建节点，使用 `node_id` 作为 key | 完整展示问题 + 选项按钮 |
| `criteria_card` | 插入购买标准摘要卡节点 | 新建节点，`node_id` = criteria_id | 对话区只展示摘要 chips，详情进底板 |
| `product_card` | 聚合到 SwipeDeck 节点 | 以 `deck_id` 为 key upsert SwipeDeck，内部按 rank 排序 | 多个 product_card 共享同一个 deck 节点 |
| `final_decision` | 插入决策摘要卡节点 | 新建节点，使用 `node_id` 作为 key | 展示 verdict + why_chips |
| `done` | 关闭 loading，冻结当前轮 | 不产生可见节点 | 触发状态机转为 Idle |
| `error` | 插入错误气泡节点 | 新建节点 | 展示 message + 重试按钮（如 retryable） |

| code | message | retryable | 前端处理 |
|------|---------|-----------|----------|

## sheet-id: SwaqBq

**前端状态机定义**

| 状态 | 触发条件 | 转换目标 | UI 表现 |
|------|---------|---------|---------|
| `Idle` | 初始状态 / done 事件到达 / 取消完成 | → Composing（用户开始输入） | 输入框可用，无 loading |
| `Composing` | 用户输入文本或选择图片 | → Streaming（点击发送） / → Idle（清空输入） | 发送按钮高亮 |
| `Streaming` | 发送请求，SSE 连接建立 | → Idle（done 到达） / → Canceled（用户停止） / → Error（错误事件） | 输入框禁用，停止按钮显示，thinking 动画 |
| `Canceled` | 用户点击停止按钮 | → Composing（用户开始新输入） | 保留已生成内容，输入框恢复 |
| `Error` | 收到 error 事件 / 连接超时 | → Streaming（重试） / → Composing（用户新输入） | 错误气泡 + 重试按钮 |
| `Clarifying` | 收到 clarification 事件 | → Streaming（用户回答后发送） | 输入框可用，placeholder 变为引导文案 |

## sheet-id: JdIO8S

**无闪烁渲染策略**

| 策略 | 实现方式 | 说明 |
|------|---------|------|
| 稳定 key | LazyColumn 的 `key` 参数使用 `node_id` | 避免 recomposition 导致列表项重建 |
| upsert 语义 | ViewModel 维护 `Map<String, ChatUiNode>`，按 node_id 更新 | thinking/text_delta 更新已有节点而非新增 |
| 批量更新 | 高频 text_delta 事件做 50ms 节流后批量更新 UI state | 避免每个 delta 触发一次 recomposition |
| SwipeDeck 增量 | product_card 到达时 append 到 deck 数据源，不重建整个 deck | deck 内部用 product_id 做 diff |
| 滚动锚定 | 新节点插入时自动滚动到底部，用户手动上滑后暂停自动滚动 | 避免用户阅读时被强制拉到底部 |
| 骨架屏限制 | 只有 thinking 使用骨架动画，criteria_card/product_card 不做预占位 | 避免占位高度与实际内容不匹配导致跳动 |

## sheet-id: hFIZKU

**错误兜底与降级策略**

| 错误场景 | 降级方案 | 用户感知 |
|----------|---------|---------|
| SSE 连接建立失败 | 重试 1 次（2 秒间隔），仍失败则显示错误气泡 | "连接失败，请检查网络后重试" |
| SSE 流中途断开 | 保留已渲染内容，显示"连接中断"+ 重试按钮 | 已有内容不丢失，可手动重试 |
| thinking 超时（>8s 无后续事件） | thinking 气泡文案改为"响应较慢，仍在等待" | 不自动中断，用户可手动停止 |
| 单个 product_card JSON 解析失败 | 跳过该卡片，继续渲染后续事件 | 用户少看到一张卡，不影响整体流程 |
| criteria_card 解析失败 | 降级为纯文本展示 criteria 信息 | 失去卡片交互，但信息不丢失 |
| 图片上传失败 | 显示上传失败提示，保留图片预览，可重试 | "图片上传失败，点击重试" |
| 后端返回 RETRIEVAL_EMPTY | 在聊天流中展示"暂无匹配商品"+ 修改标准建议 | 引导用户放宽条件 |
| 未知 event type | 静默忽略，不崩溃 | 用户无感知 |
